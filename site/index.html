Stay close to the host.

Hermes Desktop is a native Mac app for Hermes built around a direct, host-first SSH connection model. Real terminal. Real files. Real session data. No browser wrapper. No gateway API. No daemon. No extra sync layer slowly drifting away from the machine that actually matters.

Why this exists

Hermes Desktop is not trying to hide the host behind a softer abstraction. It is for the side of Hermes you want to live in on a Mac: direct SSH, canonical files, real sessions, real cron state, and a real terminal that stays honest about where your work lives.

Transport

SSH first

If ssh your-host already works from Terminal, Hermes Desktop can usually meet you there.

State model

No shadow world

No local mirror. No custom sync protocol. No silent background state that drifts away from the host over time.

User promise

Host, profile, canonical paths, session store, and scheduler source stay visible and understandable.

Scope

The app focuses on the workflows that matter when Hermes is already part of how you work: host overview, canonical file editing, sessions, usage, skills, cron, and an embedded SSH terminal with multiple tabs across hosts and profiles.

Profiles

SSH profiles

Multiple named SSH targets plus Hermes profile awareness on the same host, with state resolved against the active selection.

Terminal

Real shell

Embedded SSH terminal with tabs, appearance controls, and room for genuinely parallel host workflows without inventing a second model.

Canonical files

Conflict-aware editing

Edit USER.md, MEMORY.md, and SOUL.md on the host with remote conflict checks before save.

Sessions

Real session store

Browse and search the canonical remote SQLite store, with fallback only when the live store is unavailable.

Skills + cron

Host workflows together

Skills and cron jobs live next to the rest of the host workflow instead of being split into a separate product surface.

Packaging

One macOS build

Universal packaging for Apple Silicon and Intel from the same build flow, with release artifacts published on GitHub.

Positioning

Hermes Desktop is not competing with “more abstraction”. It is the native Mac workspace for people who want Hermes to stay direct, inspectable, and anchored to the host.

Before you download

Simple setup rule.

If this Mac can already reach the Hermes host over SSH without a password prompt or host key confirmation, the app is usually ready.

ssh your-host
  • macOS 14 or newer
  • python3 available on the Hermes host
  • Hermes data under the remote user’s ~/.hermes
Trust model

Easy to reason about.

The architecture stays legible because it does less. No helper service on the host. No gateway API requirement. No local cache that quietly becomes authoritative by accident.

That restraint is part of the product quality, not a missing feature.
